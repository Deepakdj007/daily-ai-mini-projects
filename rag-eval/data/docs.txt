RAG stands for Retrieval-Augmented Generation. It is a technique that combines a retrieval system with a language model. Instead of relying only on the model's training data, RAG first retrieves relevant documents from an external database and then generates an answer grounded in those documents. This reduces hallucinations and keeps the model's answers up to date.

ChromaDB is an open-source vector database designed for AI applications. It stores text as embedding vectors — numerical representations that capture semantic meaning. When you query ChromaDB with a question, it finds the documents whose embeddings are closest to the question's embedding, even if the exact words don't match. ChromaDB can run entirely in memory, making it ideal for prototyping.

Retrieval evaluation in RAG measures whether the system is finding the right documents. The two main metrics are context precision (how focused the retrieved results are) and context recall (whether all necessary information was retrieved). Low precision means you're pulling irrelevant documents. Low recall means you're missing important ones.

Faithfulness measures whether the generated answer is supported by the retrieved context. An unfaithful answer contains claims that cannot be found in the retrieved documents — in other words, hallucinations. RAGAS computes faithfulness by asking an evaluator LLM to verify each claim in the answer against the context.

Response relevancy measures whether the generated answer actually addresses the question that was asked. A highly faithful answer can still score low on relevancy if it answers the wrong question. RAGAS measures this by generating reverse questions from the answer and checking how similar they are to the original question.

LangChain is a framework for building LLM applications. It provides abstractions for document loaders, text splitters, vector stores, retrievers, and LLM chains. RAGAS integrates directly with LangChain, allowing you to wrap any LangChain-compatible LLM in a LangchainLLMWrapper and use it as the RAGAS evaluator.

Groq is an AI inference company that provides extremely fast LLM inference. Their API is free to use with generous rate limits, making it ideal for tutorials and small production applications. The llama-3.3-70b-versatile model on Groq is a high-quality open-source model that produces strong results for RAG generation tasks.

Embedding models convert text into vectors of floating-point numbers. Similar pieces of text produce similar vectors. The all-MiniLM-L6-v2 model from HuggingFace is a popular lightweight embedding model that runs entirely locally with no API key needed.

RAGAS is a framework for evaluating RAG pipelines and LLM applications. In version 0.2, RAGAS moved to a new API where metrics are instantiated as class instances with an explicit evaluator LLM. The old lowercase imports like faithfulness are deprecated. The correct v0.2 pattern is to use class names like Faithfulness and ResponseRelevancy, and to pass an evaluator LLM at construction time.

A test dataset for RAG evaluation consists of question-answer-context triples. The question is what the user asked. The context is the list of retrieved chunks. The answer is what the RAG pipeline generated. The reference (sometimes called ground truth) is the correct answer, written by a human or generated by a capable LLM. Context precision and context recall require a reference answer; faithfulness and response relevancy do not.
